# VOCAB BLAST! - 収録英単語データ仕様書（確定版）

本ドキュメントは、「VOCAB BLAST!」に収録される英単語データの選定基準、分類ルール、データ構造（プログラム向け定義）、および収録単語リストを定義した仕様書です。

---

## 1. 単語選定方針
プレイヤーが実用的な英語力を養えるよう、以下の基準で単語を選定しています。
* **日常会話レベル**: 日常生活や身近な場面で頻繁に使用される英単語。
* **学習指導要領対応**: 小学校、中学校、高校の英語授業・教科書で広く扱われる英単語。
* **偏りのない難易度設計**:
  * 抽象的すぎる専門用語は避け、直感的に翻訳しやすい名詞、動詞、形容詞を中心に採用。
  * タイピングゲームとしてスムーズに入力できるよう、極端に特殊な記号（ハイフンやスペースなど）を含むものは除外。

---

## 2. モード（文字数）分類ルール
ゲーム仕様書に定義された難易度モードに基づき、単語の文字数（英単語のスペル数）によって自動的または事前に以下のように分類されます。

* **【little】（7文字以下）**: 
  * タイピング初心者〜中級者向け。
  * 主に小学生〜中学校で習う基礎的な語彙、および日常生活の必須語彙が該当。
* **【long】（8文字以上）**:
  * タイピング上級者向け。
  * 主に高校生レベルの語彙、および表現力豊かな日常会話・実用語彙が該当。
* **【remix】（すべての文字数）**:
  * `little` と `long` のすべての単語がランダムに出現。

---

## 3. データ構造（Schema）

ゲーム実装時に利用する TypeScript のデータ構造を以下のように定義します。

```typescript
// 単語データの基本型定義
interface WordData {
  id: string;        // 一意の識別子 (例: "w-001")
  japanese: string;  // 画面に提示される日本語のお題 (例: "図書館")
  english: string;   // 正解の英単語 (すべて小文字で定義、例: "library")
  length: number;    // 英単語の文字数 (例: 7)
  level: 'elementary' | 'junior' | 'high' | 'daily'; // 学年・難易度分類
}
```

## 5. 実装に向けたデータ拡張ガイド
ゲーム開発のフェーズにおいて、単語数を数千件規模に増強する際は、本ドキュメントの「3. データ構造」に適合したJSONファイルを自動生成（スクリプト処理など）してアプリに読み込ませる方法を推奨します。
大文字と小文字の判定、スペースの除去、文字数カウントなどは、すべてプログラム側のロード時に自動処理させることで、メンテナンス性を向上させることができます。
